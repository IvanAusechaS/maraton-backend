Global error handling middleware Handles all unhandled errors in the application, logging useful details in development mode and restricting output in production. Differentiates between server-side (5xx) and client-side (4xx) errors to send appropriate responses.

(async) sendEmail(to, subject, text) → {Promise.<void>}

Sends an email using SendGrid API. Works perfectly with Render free tier (no SMTP port restrictions). Free tier: 100 emails/day, no domain verification needed.
Parameters:
Name Type Description
to string Recipient email address
subject string Email subject line
text string Email content in plain text format
Source:
  • utils/sendEmail.ts, line 91
Requires:
  • module:process.env.SENDGRID_API_KEY
  • module:process.env.SENDGRID_FROM_EMAIL
Throws:
Throws error if email sending fails
Type
Error
Returns:
Resolves when email is sent successfully
Type
Promise.<void>
Examples
// Send a simple email
await sendEmail(
  'user@example.com',
  'Welcome to Maraton',
  'Thank you for signing up!'
);
// Send password reset email
const resetUrl = 'https://app.com/reset/token123';
await sendEmail(
  'user@example.com',
  'Password Reset',
  `Click here to reset your password: ${resetUrl}`
);
